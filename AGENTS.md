# Уроки отладки (из эпопеи с каруселью)

- **Сначала проверяй сами ассеты, а не код.** Если визуальный эффект «не работает» на картинке — открой сам файл. У `public/templates/*/preview.webp` вбитые прозрачные скруглённые углы (альфа-канал ~16px): ни один CSS-радиус больше этого не отобразится, пока картинку не зазуммить (`scale(1.15)` на img выносит вбитый край за границы слайда).
- **Верь пользователю, а не computed styles.** Если пользователь видит баг, а тесты «в изоляции работают» — проблема в окружении/данных, а не в его глазах. Не доказывай корректность кода, воспроизводи баг end-to-end.
- **Скриншот — истина.** Для UI-багов проверяй глазами: headless Chrome (`--screenshot`, при надобности CDP для computed styles/замеров). Одна картинка заменяет десять гипотез.
- **Самописная бесконечная карусель = сначала симуляция.** Любая логика рециркуляции/спейсинга проверяется численной симуляцией (200k кадров: позиции ограничены, дрейфа нет) до внедрения. Общая перспектива (vanishing point в центре экрана) взрывает ширину дальних карточек — использовать точную проекцию с клампом ширины (×1.15 к наивной).
- **Откатывай неудачные гипотезы сразу.** Не наслаивать фиксы друг на друга: не сработало — вернуть как было и пробовать следующее.
- **Токены — на воспроизведение, не на теоретизирование.** Последовательность: воспроизвести → локализовать (бисекция, по одной переменной) → фикс → контрольный скрин. Не наоборот.

## Как читать скриншоты

- **Не судить детали по даунсэмплированному полному скрину.** Углы, радиусы, тонкие линии — только кропом в нативном разрешении (`region` в исходных пикселях файла, не в координатах показанной копии).
- **Сначала координаты элемента, потом кроп.** Не угадывать позицию угла «на глаз» — взять `getBoundingClientRect` (CDP/DevTools) и кропать по точным числам. Половина моих «багов» была кропом мимо угла.
- **Малое значение нечитаемо — проверять большим.** Радиус 20px на пёстрой картинке почти не виден; для проверки механизма временно ставить 80px. «Не вижу» ≠ «не работает».
- **Отличать клип элемента от рамки контента.** У картинки могут быть свои вбитые поля/альфа-угол: проверять контрастной подложкой (яркий фон под элемент) или чтением пикселей (`getImageData` на canvas), а не глазомером.
- **Смотреть шире искомой точки.** Соседние элементы могут перекрывать искомое (оверлап прячет углы/края) — кроп всегда с запасом контекста.
- **Анимацию не судить по одному кадру.** Снимать несколько позиций/моментов; движущийся элемент на стоп-кадре может стоять в нетипичном месте.

# Правила работы

- **Явные баги, найденные в процессе разработки — фиксить сразу**, даже если их не репортили. Не ограничиваться сообщением пользователю.
- **В финальный отчёт всегда включать репорт по багам:** какие баги были найдены и что конкретно сделано с каждым (фикс + где, либо почему отложен).
